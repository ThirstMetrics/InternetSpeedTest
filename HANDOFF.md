# SpeedTest — Handoff Document

## What This App Does

SpeedTest is a PWA that lets users test their internet speed (download, upload, ping, jitter) and plots results on a community map so others can find fast public WiFi. Geofenced to the Las Vegas metro area (~50 mi radius).

Production URL: `https://speedtest.thirstmetrics.com`

## What Changed (This Session)

All changes are **uncommitted** on the `main` branch. Four areas of work:

### 1. Backend Migration: Supabase → Neon + Auth.js

The entire Supabase dependency has been removed (both the database client and auth). Replaced with:

- **Database:** `@neondatabase/serverless` — Neon's HTTP-based PostgreSQL driver
- **Auth:** `next-auth@beta` (Auth.js v5) — Google Sign-In only, JWT session strategy

| What | Before | After |
|------|--------|-------|
| DB client | `src/lib/supabase.ts` (deleted) | `src/lib/db.ts` |
| Auth | Supabase Auth (email/password + Google) | Auth.js v5 (Google only) |
| Auth config | Inline in components | `src/lib/auth.ts` |
| Auth route | None (Supabase handled client-side) | `src/app/api/auth/[...nextauth]/route.ts` |
| History query | Direct Supabase client in TestHistory.tsx | `src/app/api/history/route.ts` (server-side) |
| Results insert | `supabase.rpc('insert_speed_test')` | `sql\`SELECT insert_speed_test(...)\`` |
| Locations query | `supabase.rpc('get_nearby_locations')` | `sql\`SELECT * FROM get_nearby_locations(...)\`` |
| Session state | Manual useEffect + onAuthStateChange | `useSession()` from next-auth/react |
| Auth modal | Email/password forms + Google button | Google button only |
| npm dep | `@supabase/supabase-js` | `@neondatabase/serverless` + `next-auth@beta` |

**Schema file:** `neon-schema.sql` — adapted from `supabase-schema.sql`:
- Added standalone `users` table (email, name, image) — Auth.js upserts on every sign-in
- Changed FK references from `auth.users(id)` → `users(id)`
- Removed RLS policies (access control at application layer now)
- Removed `SECURITY DEFINER` from functions
- Removed `handle_new_user` trigger
- Added `idx_speed_tests_user` index
- PostGIS functions `insert_speed_test` and `get_nearby_locations` are identical in logic

## Setup Steps (Before First Deploy)

### A. Neon Database

1. Create a Neon account at [neon.tech](https://neon.tech)
2. Create a new project (US-East region recommended for proximity to Cloudways NJ)
3. In the Neon SQL Editor, run:
   ```sql
   CREATE EXTENSION IF NOT EXISTS postgis;
   ```
4. Then paste and run the entire contents of `neon-schema.sql`
5. Copy the connection string into `DATABASE_URL` in `.env.local`

### B. Google OAuth

1. Go to [Google Cloud Console](https://console.cloud.google.com/) → APIs & Services → Credentials
2. Create an OAuth 2.0 Client ID (Web application)
3. Add authorized redirect URI: `https://speedtest.thirstmetrics.com/api/auth/callback/google`
4. For local dev, also add: `http://localhost:3000/api/auth/callback/google`
5. Copy Client ID and Client Secret into `.env.local`

### C. Auth Secret

```bash
openssl rand -base64 32
```
Paste the output into `NEXTAUTH_SECRET` in `.env.local`.

### D. Cloudways Deploy

Same process as before — the deploy command is unchanged:
```bash
ssh 167.71.242.157 "export PATH=/home/master/.nvm/versions/node/v20.20.0/bin:\$PATH && cd ~/applications/gahrkstjut/public_html && git pull && npm run build && pm2 restart speedtest"
```

Make sure the new env vars are set on the Cloudways server too (in the same `.env.local`).

## Android Build Steps

### Debug APK (for testing)

```bash
npx cap sync android
cd android
./gradlew assembleDebug
```

APK output: `android/app/build/outputs/apk/debug/app-debug.apk`

### Release AAB (for Play Store)

1. Generate upload signing key:
   ```bash
   keytool -genkey -v -keystore speedtest-upload.keystore -alias speedtest -keyalg RSA -keysize 2048 -validity 10000
   ```

2. Get the SHA256 fingerprint (for `assetlinks.json`):
   ```bash
   keytool -list -v -keystore speedtest-upload.keystore -alias speedtest | grep SHA256
   ```

3. Update `public/.well-known/assetlinks.json` with that fingerprint, redeploy the web app.

4. Build signed AAB:
   ```bash
   cd android
   ./gradlew bundleRelease
   ```

### Play Store Submission

- Developer account: $25 one-time at [play.google.com/console](https://play.google.com/console)
- Store listing needs: title, descriptions, screenshots (speed test, results, map, leaderboard), feature graphic
- Privacy policy URL: `https://speedtest.thirstmetrics.com/privacy`
- Content rating: expect "Everyone"
- Data safety form: collects location (approximate + precise), email, speed test results
- Review typically takes 1–7 days

## Known Risk: Google Sign-In in WebView

Google may block OAuth redirects inside Capacitor's WebView (`disallowed_useragent` policy). If this happens, install `@capacitor/browser` and open sign-in in the system browser instead. Auth.js redirect flow supports this without code changes — only the `signIn('google')` call in `AuthModal.tsx` would need to be changed to open in an external browser.

## Data Migration Note

If there is existing data in Supabase that needs to be preserved, it must be migrated manually before switching over. The Neon schema is structurally identical for `speed_tests` and `locations` tables. The `users` table is new (Supabase used `auth.users`), so user records will need to be recreated — or users simply re-authenticate via Google on their next visit (Auth.js will auto-create their record).
